## 二、逐项修复详情

### 🔴 CRITICAL #1: SQLAlchemy Enum 存储格式与查询不匹配

**文件**: `app/models/class_system.py`
**位置**: `ClassMember.role` 列定义

**问题根因**:
```python
# 修复前
role = Column(Enum(UserRole), nullable=False)

# 问题：SQLAlchemy 默认将 Python 枚举的成员名（.name）存入数据库
# 即存入 "ADMIN"、"INSTRUCTOR"、"STUDENT"（大写）
# 但查询时用 UserRole.ADMIN.value == "admin"（小写）比较 → 全部失败
```

**修复**:
```python
# 修复后
role = Column(Enum(UserRole, values_callable=lambda e: [x.value for x in e]), nullable=False)
```

**影响范围**: 所有涉及 `ClassMember.role` 的查询全部失败，包括：
- 教练查询自己的班级学员
- 教练创建班级学员文书
- 班级成员统计

**副作用**: 修复后需要数据库中 `class_members.role` 列的值与枚举 `.value` 一致（小写）。
已在数据库中执行：
```sql
UPDATE class_members SET role = 'instructor' WHERE role = 'INSTRUCTOR';
UPDATE class_members SET role = 'manager'   WHERE role = 'MANAGER';
UPDATE class_members SET role = 'student'   WHERE role = 'STUDENT';
```

### 🔴 CRITICAL #4: FastAPI 路由顺序冲突

**文件**: `app/api/student_chapters.py`

**问题根因**: FastAPI 按路由定义顺序匹配。
`/{chapter_id}` 在 `/pdf` 之前定义，导致请求 `GET /student/chapters/pdf`
被 `/{chapter_id}` 捕获，将 `"pdf"` 当作 `chapter_id`（Integer类型），
触发 FastAPI 的 `int` 类型解析错误 → HTTP 500。

**修复**: 将 `/pdf` 和 `/pdf/progress` 两个路由函数整体移到 `/{chapter_id}` 之前

```
修复前路由顺序（第144行起）:
  @router.get("/{chapter_id}")           ← 匹配了 "pdf" → 500
  @router.get("/{chapter_id}/exercises")
  @router.get("/pdf")                     ← 永远不会被匹配
  @router.post("/pdf/progress")

修复后路由顺序:
  @router.get("/pdf")                     ← 优先匹配
  @router.post("/pdf/progress")
  @router.get("/{chapter_id}")            ← 现在能正确匹配数字ID
  @router.get("/{chapter_id}/exercises")
```

**影响**: 修复前学员端 `/student/chapters/pdf` 返回 500 Internal Server Error
